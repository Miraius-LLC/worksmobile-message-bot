# コミットメッセージ規約

git commit の subject 行は **絵文字 (gitmoji) + 半角スペース + 本文** で書く。

- shortcode (`:zap:`) や Conventional Commits の type prefix (`perf:`) は使わない
- 主要な変更を表す絵文字 1 個だけ。複数並べない (`⚡️ + 🐛` ❌)
- 本文 (body) には絵文字は不要、subject 行のみ
- **`Co-Authored-By: Claude ...` フッタは付けない** (本人 push 風に揃える)
- `Merge pull request #...` 等の Git 自動生成メッセージは対象外

## メッセージスタイル

「fujimogn 本人が GitHub から push した」体裁に揃える。エージェント臭を抑える:

- subject 1 行で完結するなら **body なし** で良い
- body を書く場合は短く (目安 1〜3 行)。「自分が後で見て何をしたか分かる」レベルで十分
- 以下は避ける:
  - `## 修正` `## 検証` `## 実機結果` のようなセクション見出し羅列
  - 検証データを表形式で長文ダンプ (3 行以上の表は subject の括弧に圧縮する)
  - 「ユーザー指示: 〜」のような自分の理解を披露する書き方
- 件数 / 効果 / 通し番号は **subject の括弧** に圧縮: `(4,024 件 → 0 件)` `(58,171 件)` `(Fix A)` `(リファクタ B/C)`

## 絵文字対応表

| 絵文字 | 用途 |
|--------|------|
| ✨ | 新機能・新ジョブ追加 |
| 🐛 | バグ修正 |
| ⚡️ | パフォーマンス改善 |
| ♻️ | リファクタ (挙動変えない) |
| 🔧 | 設定ファイル変更 (config / biome / tsconfig / lefthook 等) |
| 🔨 | ビルド・ツール周り (Dockerfile / scripts) |
| 📝 | ドキュメント (README / コメント) |
| 🚀 | デプロイ関連 |
| 🔥 | 不要コード/ファイル削除 |
| 🙈 | .gitignore 変更 |
| ➕ | 依存追加 |
| ⬆️ | 依存アップデート |
| ⬇️ | 依存ダウングレード |
| 🚨 | lint/typecheck エラー修正 |
| ✅ | テスト追加・修正 |
| 🔒 | セキュリティ修正 |
| 💄 | UI/スタイル調整 |
| 🏷️ | 型定義の変更 |
| 🚚 | ファイル/ディレクトリ移動・リネーム |
| 🩹 | 軽微な修正 (typo 等) |

## 例

```
⚡️ Amazon スクレイプを並列化
🐛 PDF ダウンロード失敗時にリトライしないバグを修正
♻️ config の再帰ロードを並列化
🔨 Dockerfile から nodejs / curl を撤去
🙈 .debug/ を gitignore に追加
🔥 build/ を git 管理から外す
```
