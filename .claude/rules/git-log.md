# セッション開始時の git log 把握

新しいセッション (= 別ターミナル) で作業を始めるときは、ユーザーの最初の質問に答える前、または既存実装に関する判断を下す前に **`git log --oneline -20` を読んで最新コミットを把握する**。

## なぜ

セッション開始時にシステムプロンプトへ埋め込まれる `git status` / 直近 commit のスナップショットは、実際の HEAD よりも古い場合がある。これを鵜呑みにすると:

- 「Phase X は未着手」と誤認して既に完了した作業を再提案してしまう
- 既に実装済みの機能 (例: `--force` フラグ) を「これから実装すべき提案」として並べてしまう
- ユーザーに「もう出来ている」と何度も指摘させる無駄が発生する

## いつ確認するか

- 進捗 / 残作業 / 実装方針について質問された直後
- TODO や docs を更新する作業に着手する前
- 「既にある実装」を前提に何か提案・実装する前

差分がなければそのまま進めてよい。差分があれば最新の HEAD を信用してそれに基づいて答える。
